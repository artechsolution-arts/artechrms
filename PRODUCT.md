# Product

## Register

product

## Users

HR managers, admins, and employees at small-to-medium and enterprise companies using Artech HRMS as a multi-tenant SaaS platform. Primary users (HR managers, payroll admins) work in the system daily for people operations: hiring, onboarding, payroll runs, leave approvals, appraisal cycles. Secondary users (employees) open it occasionally for self-service tasks. Context is a desktop web browser in an office environment, during focused work hours.

## Product Purpose

Artech HRMS handles the full employee lifecycle in a single platform: recruitment pipelines, onboarding, attendance, leave management, payroll processing, performance appraisals, and HR analytics. Success means HR teams spend less time on administrative overhead and more time on people strategy. Each organization gets its own isolated environment with role-based access across SuperAdmin, HR Manager, Manager, HR User, and Employee tiers.

## Brand Personality

Professional, Clean, Efficient. The product earns trust through clarity and speed. Every screen should feel like it was designed by people who've actually worked in HR. No fluff, no decoration for decoration's sake. Data is presented, not decorated.

## Anti-references

- Old SAP / Oracle HCM: avoid the grey, dense, legacy enterprise HR software aesthetic. No nested table-in-table layouts, no modal dialogs stacked three deep, no 1990s form fields.
- Consumer apps (too casual): avoid making it feel like a mobile social app. No oversized emoji, no confetti animations, no playful rounded-blob illustrations. It needs to feel like a professional work tool.

## Design Principles

1. **Clarity over cleverness.** Every piece of information earns its screen real estate. If a label is obvious, remove it. If a control is confusing, redesign it.
2. **Speed is a feature.** HR managers run the same workflows every day. Optimize for the repeated path: fewer clicks, faster scans, keyboard-friendly.
3. **Data first, chrome last.** The content is the design. Navigation, borders, shadows exist only to organize content, never to decorate it.
4. **Role-appropriate density.** SuperAdmin and HR dashboards handle high-information density. Employee portal surfaces are simpler and more spacious.
5. **Trust through consistency.** Spacing, type scale, and color usage are predictable. Users should never have to wonder what something does.

## Accessibility & Inclusion

WCAG 2.1 AA. Sufficient contrast on all text and interactive elements. Keyboard navigation for all core workflows. No color-only state indicators (use icons or labels alongside color). Reduce-motion preference respected for transitions.
